Design

Design is an extremely broad term. Once I went to a meetup about good design, thinking it would be relevant to me, but it was about systems design. My impression from that was that it is best to keep things as separate as possible so they don't become reliant on each other. That is the extent of my understanding of systems design. Types of design I have experience with:

  • User experience design
  • Brand identity creation
  • Illustration
  • Photography

User experience design is the field where I have the most opinions and experience. If everything that impacts on user experience is a design decision, that means that user experience design has to be involved with development (for example, page speed, accessibility and device compatibility all have huge impacts on user experience), business analysis (understanding your market is deeply aligned with understanding the user of a particular service, and usable workflows may not always be compatible with what the business initially envisions), and anything that affects the context in which a service is used (such as marketing).

I see user experience design as a collaborative endeavour, building shared understanding amongst everyone who makes decisions that impact on the user experience. It involves user research and testing, and it involves bringing together different parts of the team who build a product. The visible part is using the understanding from users and the team to create user personas, journeys and wireframes. The bulk of the work involved with getting those artifacts right is talking to people, facilitating workshops, and understanding and negotiating user and business needs.

I've run workshops, made prototypes from paper to code, and run usability testing sessions with end users. I write nice, maintainable front-end code bases with object-oriented CSS and semantic HTML5 markup. I strongly believe that HTML and CSS can be a powerful design tool, and for web apps I advocate a workflow of creating modular style guides rather than building page-by-page high-fidelity designs. Good CSS architecture can be reflected in a consistent experience that is easier to use and less painful to maintain. Pre-processors like SASS help fill in some of the gaps of CSS as a language.
